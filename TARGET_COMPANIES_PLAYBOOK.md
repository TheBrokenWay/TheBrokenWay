# Target companies playbook

This document answers a narrower question than general traction:

How does Tillar Technologies get noticed by NVIDIA, Microsoft, Meta, OpenAI,
and similar companies without giving away the moat?

## Core rule

Large platform companies do not usually notice founders because of private
complexity.

They notice founders because of visible signal in one or more of these buckets:

1. strong public thesis
2. clear platform relevance
3. proof artifacts others can evaluate quickly
4. alignment with their current programs or narratives
5. early user or pilot evidence

## What gets attention from this class of company

The reliable signals are:

- a live public demo
- clear GitHub profile and repo surfaces
- a repeatable technical narrative
- evidence of a real market or regulated workflow problem
- usage of their ecosystem in a way that flatters their strategy
- founder persistence and shipping cadence

## The message each target should hear

You should not send the same message to all four.

## NVIDIA message

"We are building governed AI infrastructure for high-trust workflows. The
public demo proves the control pattern; the private layer applies it to real
operational boundaries."

## Microsoft message

"We are building governance infrastructure that helps organizations deploy
capable AI systems with bounded control, reviewability, and better enterprise
oversight."

## Meta message

"We are contributing a public governance thesis and proof surface to the open
AI ecosystem while keeping the private enforcement mechanics protected."

## OpenAI message

"We are working on governed deployment patterns for capable agents, with public
proof artifacts and a private control layer designed for high-trust use cases."

## What not to do

Do not:

- pitch as a vague AGI philosophy project
- bury the governance claim under too much theory
- lead with private complexity that cannot be verified
- wait until every subsystem is finished before reaching out

## What to do instead

Lead with:

- one thesis
- one proof repo
- one demo path
- one lineage page
- one founder story
- one enterprise relevance sentence

## Concrete 30-day targeting sequence

### Week 1

- finalize public GitHub profile surfaces
- keep demo repo current
- create social preview assets
- tighten website copy to match the new GitHub framing

### Week 2

- apply to NVIDIA Inception
- apply to Microsoft for Startups
- join OpenAI startup community
- prepare a short outreach packet for partner and developer-relations contacts

### Week 3

- publish one clean technical post or demo thread on the governance thesis
- publish one short founder-origin post tied to real product need
- point every post back to one public proof surface

### Week 4

- reach out selectively to ecosystem and startup program contacts
- use the public materials as the entry point
- ask for feedback, program fit, or technical conversation, not generic
  validation

## Success criteria

The goal is not instant partnership.

The goal is to make you legible and hard to dismiss.

For these companies, that usually means:

- they can understand the category you belong to
- they can see why it matters
- they can verify that you ship
- they can place you inside one of their existing strategic frames

That is how you get noticed without giving away the moat.
